# Навигационная ментальная карта — панель управления жизнью

Персональное приложение (PWA) для управления повседневностью через связь с
миссией и глобальными целями. Карта декомпозирует **миссию → роли → направления →
проекты и операции → задачи**, помогает находить возможности и держать баланс
видов деятельности.

Работает **в браузере и оффлайн**, устанавливается на телефон, данные хранятся
**на устройстве**. Бесплатно, без сервера и регистрации.

## Зачем это

1. **Управление повседневностью** — «панель управления жизнью» (экран «Панель»).
2. **Связь с миссией** — у каждой активности видна цепочка вверх до миссии.
3. **Выявление возможностей** — неочевидные пути к целям (метка ✦ «Возможность»).
4. **Баланс деятельности** — экран «Баланс» показывает перекосы по сферам и осям.

## Структура карты (7 уровней)

1. Миссия и расшифровка
2. Принципы по сферам
3. Роли
4. Направления
5. Проекты и операции (мастер-список, 8 осей классификации)
6. Динамическая приоритизация (фокус недели, режим ПУ/БУ, статусы, квадрат)
7. Statement of Work проектов (объём, результаты, задачи)

**Сферы** (Личная / Социальная / Карьерная) — сквозной фильтр всех уровней.

**8 осей классификации активности:** энергия ПУ/БУ · тип влияния (прямое /
синергетическое / поддерживающее / восстанавливающее) · сфера · статус (начато /
в ожидании / завершено) · квадрат срочность-важность (1–4) · категория
(обучающее / созидающее / поддерживающее / исследовательское) · горизонт
(кратко/средне/долгосрочное/постоянное) · происхождение (self-made / возможность).

Карта **предзаполнена** синтезом твоих журналов и полностью редактируется.

## Запуск (для разработки)

```bash
npm install
npm run dev      # открой напечатанный адрес (http://localhost:5173)
```

Сборка и предпросмотр продакшн-версии (PWA):

```bash
npm run build
npm run preview
```

## Установка на телефон

После публикации (см. ниже) открой адрес приложения в браузере телефона →
меню → «Добавить на главный экран». Дальше работает как обычное приложение, в
том числе оффлайн.

## Хранение и резервная копия

Данные хранятся в браузере устройства (localStorage). Чтобы не потерять их:

- Меню → **«Скачать JSON»** — сохраняет файл карты.
- **«Загрузить»** — восстанавливает карту из файла (перенос между устройствами).
- **«Сброс»** — возврат к стартовому шаблону.

Файл можно при желании коммитить в этот репозиторий вручную — git сохранит
историю изменений.

## Публикация (GitHub Pages, бесплатно)

В репозитории настроен авто-деплой: при пуше в ветку `main` GitHub Actions
собирает приложение и публикует на GitHub Pages.

Один раз включи Pages: **Settings → Pages → Build and deployment → Source:
GitHub Actions**. Адрес приложения: `https://<логин>.github.io/gmm-pub/`.

> Базовый путь задан как `/gmm-pub/` в `vite.config.ts`. Если имя репозитория
> другое — поправь `base`.

## ИИ-координатор

Кнопки ИИ (поиск возможностей, декомпозиция, рекомендации, чат) присутствуют и
помечены «скоро» — архитектура под них готова (`src/services/ai.ts`).
Реализация запланирована следующим этапом (Claude API, ключ хранится на
устройстве).

## Технологии

Preact + TypeScript + Vite, `@preact/signals`, vite-plugin-pwa. Лёгкий бандл,
рассчитан на слабые устройства. Без бэкенда и внешних сервисов.
